Elastic Beanstalk　・・・　アプリケーションを迅速位デプロイし管理するサービス

OpsWorks　・・・　Chef や Puppet のマネージド型インスタンスを利用した構成管理サービス
Puppet、Chef、　・・・　コードを使用してサーバーの構成を自動化するためのオートメーションプラットフォーム

CodeDeploy　・・・　EC2、Lambdaなどのさまざまなコンピューティングサービスへのソフトウェアのデプロイを自動化する、フルマネージド型のサービス

Fargate　・・・　EC2やEKSなどで動作するコンテナ向けのサーバレスコンピューティングエンジン


Snowball edge　・・・　データ移行サービス

Snowmobile　・・・　Snowball edgeの大容量向け

Redshift　・・・　データウェアハウス（様々なシステムからデータを集める場所）サービス

AWSマーケットプレイス　・・・　AWS上で利用可能なソフトウェアやサービスを購入することができる購入することができるオンラインストア

Dynamo DB　・・・　NoSQLデータベース。セッションデータの処理に向いていて、高速で処理を実行できる。
　　　　　　　　　　 2つのモードがあり、オンでマウンドとプロビジョニング（容量が予測できる時）
　　　　　　　　　　JSON形式のデータの格納などができる

DynamoDB Accelerator (DAX)　・・・　DynamoDB用に特化し、レイテンシをマイクロ秒単位まで低下させるキャッシュ

STS（Security Token Service）　・・・　一時的な限定権限認証情報を取得できます

Key Management Service (KMS)　・・・　暗号キーを作成して管理し、様々なサービスでの使用を制御する

Elastic Container Registry (ECR)　・・・　DockerコンテナイメージをAWSクラウドに保存・管理することができる

EFS　・・・　EC2から接続できるスケーラブルなファイルストレージ。
　　　　　　　複数インスタンス間で共有することができる。

CloudFormation　・・・　インフラ構築をコード化して自動セットアップを支援するサービス

Well-Architected Tool　・・・　アーキテクチャ設計原則に沿っているかをチェックし、ベストプラクティスと比較できるサービス

Glacier Deep Archive　・・・　Glacierよりも安くデータの保存ができるが、取り出しがさらに遅くなる

Amazon Aurora　・・・　高速処理が可能なリレーショナルデータベース

AMS Budgets　・・・　予算を設定し、予算を超えた時にアラート通知できる

TAM　・・・　テクニカルアカウントマネージャーの略。WS技術に関する様々なアドバイスや。ベストプラクティスに沿った計画、運用をサポートする。

EBSのスナップショットを作ると、スナップショットはそのEBSのリージョンにそのまま保存される

Amazon Connect　・・・　クラウド上にコールセンターシステムを作れる

規模の経済　・・・　AWSが成長すればするほど、利用者はサービスを安く受けられること

AWS Config　・・・　設定変更履歴を記録する。企業ポリシーやコンプライアンスに一致しているか監視する。
　　　　　　　　　　　ルールから逸脱したリソース設定などを検知すると自動でSNSがトリガーされる


リザーブドインスタンスを利用することで前払いができる

RDSからAuroraへは、スナップショットを使って移行する

AWS System Manager　・・・　AWSリソースのグループ化、EC2の自動化、オンプレミス環境と統合して管理ができる。
　　　　　　　　　　　　　　　　運用データの確認、運用の自動化

NATゲートウェイはプライゲートサブネットからインターネットへのアクセスを可能にするが、設置場所はパブリックサブネット

リードレプリカはRDSの追加インスタンスで、コピーを作成できる。レポート処理などに使用できる。

Savings Plans　・・・　EC2の購入オプションで、1年、3年の間に一定の使用を約束することで割引される。

AWS Kinesis Data Streams　・・・　リアルタイムのデータストリーミングサービス

AWS Storage Gateway　・・・　

AWS Application Discovery Service　・・・　オンプレミスのデータセンターの情報を収集し、awsへ移行を支援するサービス

AWS Server Migration Service　・・・　データベースではなくサーバーの移行を支援する

Amazon Athena　・・・　S3ないのデータの分析

コンシェルジュチーム　・・・　請求、アカウントのベストプラクティスを提案してくれるチーム

プロフェッショナルサービス　・・・　クラウドで目標のビジネス効果を達成するためのサポートチーム

EFSは複数のインスタンスのストレージとして使える

RDS,RedShiftには自動バックアップ機能がデフォルトでついている

S3 One Zone-IA　・・・　長期間、あまりアクセスされない、重要度の低いデータの保存。バックアップなどでよく使われる

S3 Standard-IA　・・・　長期間、あまりアクセスされない、重要度の低いデータの保存。バックアップなどでよく使われる

S3 Intelligent-Tiering　・・・　アクセスパターンが変化、不明なデータの保存

AWS X-Ray　・・・　アプリケーションの問題点やエラーの原因を特定する。

Amazon Keyspaces　・・・　DynamoDBとは違い、オープンソースに互換性があるNoSQL型のデータベース

SCP　・・・　組織(Organization)ごとにアクセスを管理するポリシー

AWS Pay-As-You-Go　・・・　使った分だけライセンス料を支払うこと

ルートアカウントは全ての権限があるため、アクセスキーがある場合は削除する

インラインポリシー　・・・　ユーザー自身で作成、管理されるポリシー。プリンシバルエンティティ（ユーザー、グループ、ロール）にアタッチされる。

Access Advisor　・・・　IAMユーザーが最後にアクセスした日付や時刻などを表示できる。

Auto-Scalingではデフォルトで最古の軌道設定のインスタンから削除される

パワーユーザー　・・・　IAMの操作権以外のアクセス権限を持つユーザー

Direct Connect　・・・　異なるリージョン間を接続するサービス

IAM Access Analyzer　・・・　外部エンティティと接続されているS3バケットなどの不正アクセスを検知することができる。

AWS Directory Service　・・・　ユーザーに関わる各情報を管理してユーザー認証を実装する

サブネットはAZを跨いで設置できない

スタンダードタイプのリザーブドインスタンスはマーケットプレイスで販売できる。前払い金額の設定もできる。
