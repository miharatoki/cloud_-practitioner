AWSにとってセキュリティは最優先事項
AWS側もかなり高度なセキュリティを敷いているが、ユーザが意識してセキュリティを高めたければいけない

責任共有モデル　・・・　ユーザとAWSが責任を共有して持つモデル

AWSの責任　・・・　クラウド本体のセキュリティ
・リージョンやアベイラビリティ、エッジロケーションなどのデータセンター
・各サービスのソフトウェア
・セキュリティ保護プログラム
・データセンターのセキュリティ
・ハイパーバイザー（仮想化ソフトウェア）のセキュリティ

ユーザの責任　・・・　クラウド内のセキュリティ
・ユーザのアプリのセキュリティ
・セキュリティグループ、ファイアウォール構成


IAM・・・　クラウドリソースへのアクセス管理サービス。IAMユーザーやIAMグループを作成し、権限を管理する
　　　　　　必要最低限の権限に抑える
　　　　　　許可と拒否が相反するときは拒否が優先される

APIキー　・・・　アクセスキーとシークレットアクセスキーのペアで構成される。
　　　　　　　　　プログラムの認証やコマンドラインの操作に使う。
　　　　　　　　　現在は非推奨。代わりにIAMロールを使用する。

IAMロール　・・・　EC2やLambdaなどのAWSリソースに対して権限を付与できる。
　　　　　　　　　　IAMロールとリソースを直接紐づけることができるので、キーを管理する必要がない。


AWS Shield ・・・　 DDos攻撃に対する保護サービス
　　　　　　　　　　　スタンダード（無料）とアドバンス（有料）の２つがあり、アドバンスではAWS WAFを無償で無制限に使用できる
　　　　　　　　　　　Cloud Frondで自動適用される

AWS WAF ・・・　Web Application Firewall の略。一般的なWebの脆弱性からアプリケーションを守る、マネージド型のファイアーウォール。
　　　　　　　　　データの中身をアプリケーションレベルで解析できるので、アプリケーションにセキュリティ上の問題があっても、それを無害化できる。
　　　　　　　　　基本無料だが、カスタマイズしたセキュリティルールの指定に基づいて課金される

DDos攻撃　・・・　攻撃者が複数のコンピュータを乗っ取り、一斉に行う攻撃


Amazon Inspector ・・・　EC2上にデプロイされたアプリケーションのセキュリティとコンプライアンスを向上させるための、脆弱性診断を自動で行うサービス

AWS Artifact ・・・　コンプライアンス関連情報を見れるサービス

AWS Key Management Service(KMA) ・・・　AWS上で暗号化キーを作成、管理し、AWSのサービス、アプリケーションでの使用を制御するサービス
